  - /api/run-automations  → Every 6 hours (0, 6, 12, 18 UTC)
  - /api/update-segments  → Daily 5 AM UTC

Lead Lifecycle:
  new_lead → initial_sequence → nurture → hot_lead → converted
                                                  ↘ lost

API Endpoints:
  POST /api/save-automation - Save new automation
  POST /api/seed-automations - Create pre-built automations
  POST /api/run-automations - Execute automations (cron)
  POST /api/update-segments - Detect segments (cron)
  POST /api/openphone-webhook - Receive SMS/call events

WHAT HAPPENS AFTER DEPLOYMENT:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✅ Forms Submit:
  → Lead created with segment="new_lead"
  → lead-capture.js sends initial SMS + email quote
  → Marked as quote_sent_at

✅ Every 6 hours (cron runs):
  → /api/run-automations executes
  → Checks for matching leads
  → Sends follow-up SMS/email based on automations
  → Logs execution in lead_automation_runs

✅ Lead responds (SMS/call):
  → OpenPhone webhook fires
  → Updates response_count++, last_responded_at
  → Moves to hot_lead segment
  → Automations pause (manual follow-up)

✅ Daily at 5 AM (cron runs):
  → /api/update-segments executes
  → Moves initial_sequence → nurture (7+ days no response)
  → Detects one_time customers → reengagement
  → Detects winback candidates

✅ Lead books:
  → Converts to customer
  → Segment = "converted"
  → Booking tracking starts
  → Re-engagement automations trigger after 3 days
